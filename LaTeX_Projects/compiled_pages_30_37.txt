2.3.3
Application Architecture
We built the codebase inside a monorepo containing three applications.
The backend REST API runs on Node.js [1] and Sails.js [2], communicat-
ing with MongoDB [3] using the sails-mongo driver. The web dashboard
is built with React [4], Vite [5], and TypeScript [6], using Redux Toolkit [7]
to manage app state. The mobile client utilizes React Native [9] and
Expo [8], handles QR code scanning using the camera API [12], and
navigates screens via React Navigation.

Chapter 3

3.1
Introduction
We began development by implementing the platform’s core security and
account management. This chapter details our implementation of Re-
lease 1. Our main objective was setting up a secure, role-isolated ten-
ant boundary so different accounts can access only their specific brand
catalogs.
Our implementation focused on three primary areas:
• Token-based Logins: We built a stateless authentication system
using JSON Web Tokens (JWT [10]) and integrated Google OAuth
2.0 [19].
• Role-Based Permissions: We defined concrete privileges for Clients,
Technicians, Company Admins, and Super Admins.
• Profile Forms: We created custom profile views to collect basic
user info and technical credentials.
